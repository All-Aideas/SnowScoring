Por qué se llama Snow·Score

Como la nieve: un copo aislado no construye nada — la capa acumulada en el tiempo, sí. Cada boleta, cobro o recibo es un copo. Una sola, da poca información. Repetida durante 6, 9, 12 meses forma una capa imposible de fingir.

Por eso cada datapoint se pondera por su antigüedad y por su recurrencia: un alquiler pagado 12 meses seguidos pesa exponencialmente más que un comprobante puntual. El sistema no premia el volumen — premia la consistencia sostenida en el tiempo. Por eso una persona "sin historial" del buró tradicional puede tener un SnowScore A en 6 meses de uso normal: la evidencia siempre estuvo ahí, solo que nadie la leía como patrón.

— Preguntas frecuentes (institucional)

Lo que el área de riesgo
siempre nos pregunta.

¿Cómo manejan la privacidad del usuario y el consentimiento?

El usuario es dueño de sus datos. SnowScore procesa los documentos y guarda únicamente el score y un hash criptográfico (keccak256) on-chain. Nunca exponemos tickets, montos ni merchants a terceros. Cada consulta de la API requiere consentimiento explícito vía wallet del usuario.

¿Reemplaza al proceso de KYC tradicional?

No. SnowScore es una capa complementaria al KYC: aporta señales de comportamiento financiero (recurrencia, estabilidad, capacidad de pago) que el KYC no captura. Se integra como input adicional al motor de decisión de crédito, no como sustituto del compliance regulatorio.

¿Qué pasa con las regulaciones (BCRA, CNV, CNBV)?

Trabajamos sobre Avalanche C-Chain con anchoring de hashes (no PII on-chain). El modelo de scoring es explicable y auditable: cada score viene con su breakdown de señales. Roadmap incluye certificación bajo marcos regulatorios locales por país (Argentina, México, Colombia, Brasil).

¿Cómo se integra técnicamente con nuestro core bancario?

API REST estándar (Bearer auth, JSON). El endpoint POST /v1/score devuelve score + grade + confidence + on_chain_proof. Latencia objetivo < 500ms para lookups, < 3s para análisis nuevo. SDK oficial para Node, Python y Java en roadmap post-hackathon.

¿Por qué Avalanche y no otra blockchain?

Finalidad sub-segundo (UX equivalente a una API web2), costos por anchor < $0.01 (viable para LATAM), y la posibilidad de migrar a una L1 dedicada para identidad financiera. Además, el ecosistema institucional de Avalanche en LatAm hace al stack defendible frente a compliance.

¿También procesan ingresos en cripto y remesas?

Sí, como un canal de entrada más — no es el foco del producto, pero el caso es real: programadores y freelancers LATAM que cobran en USDC/USDT, o gente que recibe remesas en cripto, no tienen forma de mostrarle ese ingreso al banco. SnowScore acepta hashes de transacciones y wallets como evidencia adicional, las normaliza al mismo schema que un ticket o un extracto, y suma esa señal al score. Para el banco es un input más; para el usuario, deja de ser un ingreso invisible.

¿El usuario necesita una wallet cripto para tener SnowScore?

No. SnowScore funciona sin wallet. El producto core es el score derivado del análisis de los documentos del usuario, y se consume vía API REST como cualquier servicio web2. La wallet es un add-on opcional que aporta dos cosas: (a) portabilidad — el usuario es dueño de la prueba criptográfica de su reputación y puede llevarla a cualquier fintech sin pedírnosla — y (b) verificación third-party: cualquiera puede recomputar el hash y comprobarlo en Snowtrace sin trust en SnowScore. Para el grueso de usuarios LATAM no bancarizados, el score funciona desde el primer documento subido — la wallet llega después, si y cuando quieran.

¿Cómo se pondera el tiempo en el cálculo del score?

El SnowScore se construye sobre 3 dimensiones, todas function of time: recurrencia (cuántas veces se repite el mismo patrón de pago), estabilidad (cuán constante es el monto/fecha mes a mes) y consistencia (cuántos meses seguidos sin fallar). Cada datapoint se pondera por antigüedad y por cuántas veces se repitió: un alquiler pagado 12 meses seguidos puede pesar 30–50× más que un comprobante puntual. Por eso se llama Snow·Score — como la nieve, lo que vale es la capa acumulada, no el copo aislado. Un usuario sin historial bancario puede levantar un grade A en 6 meses de uso normal: la evidencia siempre estuvo ahí, solo que nadie la leía como patrón.
